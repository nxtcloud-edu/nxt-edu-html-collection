# WO-018: 기업인턴십 코호트 추가 + 분류 '웹페이지' 확장 + 팀 선택
상태: 완료 (2026-07-13, 검증자 Claude — 계약·검증·회귀 실측 통과, main 머지)
작성: Claude (Planner) / 실행: Hermes (Coder)
워크트리 게이트: `wo/018` 브랜치

## 목표
(1) 코호트 `2026-고대세종-기업인턴십` 추가 — 이 코호트는 이름 대신 **팀(1팀~8팀)** 을 쓴다.
(2) 분류 `랜딩페이지`를 `웹페이지`로 개명·확장 (아이디어 노트 등 문서형 커버).
근거: `docs/planning/DECISIONS.md` 2026-07-13. 시딩(8개 팀 파일 업로드)은 검증자 몫.

## 설계 결정 (변경 금지)
1. **코호트**: COHORTS에 `'2026-고대세종-기업인턴십'` 추가 (기존 2개 유지).
2. **팀 규칙** (server.js SSOT): `TEAM_COHORTS = { '2026-고대세종-기업인턴십': ['1팀',…,'8팀'] }`.
   업로드 검증: affiliation이 TEAM_COHORTS에 있으면 name이 해당 팀 목록에 없을 때
   400 `"팀을 선택하세요."` (일반 코호트는 기존 이름 규칙 유지).
   `GET /api/cohorts` 응답을 `{ cohorts: [{ name, teams: [...]|null }] }`로 확장하되
   **하위 호환**: 기존 배열 형태를 쓰는 UI가 있다면 UI도 함께 갱신 (계약 단절 금지 — 한 커밋에서).
3. **분류 개명**: CATEGORIES = `['미니게임', '웹페이지']`. `normalizeCategory`가 레거시 값
   `'랜딩페이지'`를 `'웹페이지'`로 매핑 (기존 저장 데이터 무수정 호환 — 오하린·한지우 건).
   UI 탭·업로드 select가 API 기반이면 자동 반영 확인, 하드코딩이면 갱신.
4. **upload.html**: 코호트 선택이 팀 코호트면 "이름" 필드를 **팀 select(1팀~8팀)** 로 전환
   (라벨도 "팀"으로), 일반 코호트면 기존 텍스트 입력. 전환 시 값 초기화.
5. **README**: 4장 문구에서 랜딩페이지 → 웹페이지 반영, 팀 수업은 팀 선택이라는 한 줄.
6. **커밋 분리** (최소 2): ① feat: 서버(코호트·팀 검증·분류 개명·API)+테스트 ② feat: UI+README.
