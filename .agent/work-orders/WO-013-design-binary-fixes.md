# WO-013: 라이트 모드 카드 디자인·간격·로고 바이너리 수정
상태: 대기
작성: Claude (Planner) / 실행: Hermes (Coder)
워크트리 게이트: `wo/013` 브랜치 (README 규칙)

## 목표
사용자 리포트 3건 + 백로그 1건 수정. 프로덕션 실측으로 전부 재현 확인됨.

## 수정 항목 (근거 포함)

### 1. 로고 바이너리 손상 (원인 확정 — 최우선)
프로덕션에서 로고가 깨진 이미지로 렌더됨. 원인: serverless-http가 바이너리 응답을
base64 설정 없이 처리해 PNG 바이트 손상 (실측: 원본 106,033B/md5 a00bf1… vs 프로덕션 응답
191,767B/md5 4e7b8c…, `file` 판정 "data"). 로컬 node 서빙은 정상 — Lambda 경로만 손상.
- **수정**: `lambda.js`에서 `serverless(createApp(), { binary: ['image/*'] })` 방식으로
  바이너리 콘텐츠 타입 지정. **이 WO에 한해 lambda.js 수정 허용** (이 파일이 수정 대상).
- 검증: DRY_RUN으로는 재현 불가 — 코드 리뷰 수준으로 옵션 정확성 확인하고, 프로덕션 바이트
  검증은 검증자(배포 후 md5 대조)가 수행한다고 TURN_LOG에 명시.

### 2. 라이트 모드 카드 가시성 (사용자 리포트)
배경(#f5f6fa)과 카드(흰색)의 대비가 없고 테두리가 연해 카드 경계가 거의 안 보임 —
수업별 모아보기·콘텐츠 카드 모두. 다크 모드는 문제없음.
- **수정 방향** (라이트 모드 한정 오버라이드):
  - 카드 테두리를 인지 가능한 대비로 (`#c9cede` 수준), 은은한 그림자
    (`0 1px 3px rgba(28,31,46,.08), 0 4px 14px rgba(28,31,46,.06)` 감각)
  - hover 시 테두리 강조 + 그림자 상승 (인터랙티브임이 드러나게)
  - 필터 탭·정렬 버튼의 활성/비활성 대비도 함께 점검 (활성 상태가 뚜렷하게)
  - 다크 모드 기존 룩은 유지

### 3. 분류 칩–좋아요 수 간격 (사용자 리포트)
콘텐츠 카드에서 분류 칩(미니게임/랜딩페이지 박스)과 `♥ N`이 간격 없이 붙어 있음.
- **수정**: 칩과 좋아요 묶음 사이 `gap: 10px` 수준의 간격. 하트·숫자를 한 단위로 묶어 정렬
  (수직 정렬도 칩과 맞출 것). cohort.html 카드도 동일 구조면 함께.

### 4. 업로드 페이지 카피 정합 (백로그)
분류에 랜딩페이지가 생겼는데 upload.html이 아직 "내 게임 업로드"/"PUBLISH YOUR GAME"/"HTML 게임".
- **수정**: "내 콘텐츠 업로드" / "PUBLISH YOUR CONTENT" / "HTML 콘텐츠" 로 통일.
  index.html 네비의 "내 콘텐츠 업로드" 버튼 문구와 일치 확인.
