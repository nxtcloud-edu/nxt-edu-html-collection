# WO-023: 홈 탭 분리 + 페이지네이션 + 카피·칩 대비 수정
상태: 검증 대기
작성: Claude (Planner) / 실행: Hermes (Coder)
워크트리 게이트: `wo/023` 브랜치

## 목표
사용자 리포트 3건: (1) 히어로 카피 줄바꿈, (2) 라이트 모드 분류 칩 글씨 흐릿,
(3) 홈을 "수업별 모아보기 / 콘텐츠 둘러보기" 탭으로 분리하고 콘텐츠는 10개 페이지네이션.
DESIGN.md(§5 간격 스케일·§2 색상) 준수.

## 설계 결정 (변경 금지)
1. **히어로 카피** (index.html): 두 문장을 줄바꿈으로 분리하고 현행 동작에 맞게 정확화 —
   1줄: "AI 리터러시 수업에서 수강생들이 자기 AI 에이전트와 협업해 만든 웹페이지와 미니게임입니다."
   2줄: "카드를 누르면 사이트 안에서 바로 보고, 피드백과 추천을 남길 수 있어요."
   (기존 "새 탭에서"는 내부 뷰어 도입 이후 사실과 다름 — 랜딩페이지→웹페이지도 반영)
2. **분류 칩 대비 (라이트)**: `.category` 칩 텍스트·보더를 뚜렷하게 — 텍스트 `#3f4560` 이상
   (배경 흰색 대비 WCAG AA 4.5:1 충족), 보더 `#b7bdd1` 수준. 다크 모드는 현행 유지.
3. **홈 탭 구조** (index.html):
   - 히어로 아래 대형 탭 2개: **콘텐츠 둘러보기**(기본) | **수업별 모아보기**
   - 탭 상태는 URL 해시(`#classes`)로 — 새로고침·공유·뒤로가기에서 유지, 해시 없으면 콘텐츠 탭
   - 탭 전환 시 스크롤 위치 유지, 기존 정렬·분류 필터는 콘텐츠 탭 내부로
   - 탭 스타일은 기존 filter 활성 문법(시안 보더+연시안 배경) 계승하되 한 단계 크게
4. **페이지네이션** (콘텐츠 탭): 페이지당 10개, 클라이언트 사이드(/api/games 전체 응답 기준).
   하단에 `← 이전 / 1 2 3 … / 다음 →` (현재 페이지 강조, mono). 정렬·분류 변경 시 1페이지로 리셋.
   전체 카운트("N개의 콘텐츠")는 필터 적용 후 총 건수 유지.
5. cohort.html·upload.html·view.html은 카피·칩 대비 공통 부분만(칩 스타일은 theme.css SSOT).
6. **커밋 분리** (최소 2): ① feat: 탭 구조+페이지네이션 ② style·docs: 카피·칩 대비.

## 작업 단계
1. index.html 탭·페이지네이션 (DESIGN.md 스케일 변수만 사용)
2. theme.css 칩 대비(라이트 한정), 히어로 카피
3. DRY_RUN 실측: 12건 이상 시딩해 2페이지 이동·필터 리셋·해시 유지·라이트 칩 가독 확인
4. npm test 그린, TURN_LOG 완료 헤더 + wo/023 커밋
