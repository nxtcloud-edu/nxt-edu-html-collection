# WO-012: nxtcloud 로고 적용 + 라이트/다크 테마 토글 (기본 라이트)
상태: 대기
작성: Claude (Planner) / 실행: Hermes (Coder)
워크트리 게이트: `wo/012` 브랜치 (README 규칙)

## 목표
사이트에 NXT Cloud 브랜딩(로고)을 적용하고, 라이트/다크 테마 토글을 붙인다. **기본은 라이트 모드.**
근거: 사용자 확정 (2026-07-12).

## 설계 결정 (변경 금지)
1. **로고**: `/Users/glen/Desktop/nxtcloud-logo.png`를 `html-delivery/public/assets/nxtcloud-logo.png`
   로 복사해 커밋 (106KB, 흰 바탕 + 파랑/마젠타 심볼 + 검정 워드마크).
   - 네비 좌측 로고로 사용 (기존 `</>` 글리프 대체), 클릭 시 `/` 이동. CSS로 높이 ~32px.
   - 흰 바탕 이미지이므로 다크 모드에서 자연스러운 칩으로 보이게 `border-radius` 소폭 적용.
   - favicon으로도 연결 (`<link rel="icon" href="/assets/nxtcloud-logo.png">`) — 4개 페이지 전부.
2. **테마 토글**:
   - 현행 다크 팔레트를 `[data-theme="dark"]`로 옮기고, `:root` 기본을 **새 라이트 팔레트**로.
     라이트: 배경 `#f5f6fa`, 패널 `#ffffff`, 테두리 `#d9dce8`, 본문 `#1c1f2e`, 보조 `#5b6178`,
     포인트 색은 브랜드 유지(`#e94560`, 링크·강조는 라이트에서 대비 확보를 위해 `#0092c8` 계열로 조정 가능).
   - 네비 우측에 토글 버튼(라이트에서 "🌙 다크", 다크에서 "☀️ 라이트" — 텍스트+이모지).
   - 선택은 `localStorage('theme')` 저장, **저장값 없으면 라이트** (prefers-color-scheme 무시 — 사용자 확정).
   - FOUC 방지: `<head>` 최상단 인라인 스크립트로 data-theme을 렌더 전에 설정.
   - 공통 스크립트/스타일은 중복 4벌 대신 `assets/theme.js`·공통 CSS 변수 블록으로 정리해도 좋다
     (단일 SSOT 권장 — 4개 페이지 인라인 중복 금지).
3. **적용 범위**: index.html, cohort.html, upload.html, view.html 4개 전부. 두 테마 모두에서
   텍스트 대비(WCAG AA 감각)와 카드·버튼·입력 필드 가독성 확보.
4. **iframe 콘텐츠(수강생 게임)는 테마 영향 없음** — 그대로 둔다.
5. **커밋 분리** (최소 2): ① `feat:` 테마 시스템+토글 ② `feat:` 로고·파비콘 적용
   (assets 바이너리는 ②에 포함).
