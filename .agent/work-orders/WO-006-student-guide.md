워크트리 게이트: `wo/006` 브랜치 (README 규칙)

## 목표
game-ver1.html을 받아가는 수강생을 위한 안내 문서. 퍼블릭 레포의 첫 화면인 **루트 README.md**로
작성한다 (현재 루트 README 없음). 수강생이 이 문서만 보고 "게임 받기 → 자기 AI 에이전트와 협업해
개선 → 업로드·배포 → 실유저 피드백 → 재개선" 사이클을 완주할 수 있어야 한다.
근거: `docs/planning/DECISIONS.md` — 프로젝트 목표 = 이 사이클의 체험.

## 설계 결정 (변경 금지)
1. **독자는 비개발자 수강생** — 전문용어 최소화, 단계별 명령·클릭 수준으로 구체적으로. 한국어.
2. **구성** (이 순서, 각 섹션 간결하게):
   ① 이 레포는 무엇인가 (한 문단 — AI와 협업하는 빠른 기능 개발 체험)
   ② 게임 받기 — box-game/run-game 중 선택, 파일 하나를 내려받아 브라우저로 여는 법
   ③ AI 에이전트와 개선하기 — CONFIG 블록 소개 + "AI에게 이렇게 요청해보세요" 예시 프롬프트
      3~4개 (쉬운 것→어려운 것: 색·속도 바꾸기 → 최고점수 저장 → 난이도 상승 → 새 장애물 종류).
      알려진 개선 여지(시작 전 입력 오류, 빠른 화면에서 2배속 등)를 "찾아서 고쳐보세요" 힌트로 소개
   ④ 배포하기 — 업로드 페이지(강사가 수업에서 공지하는 URL)에서 소속·이름·html 업로드 →
      발급 URL 확인. **URL은 하드코딩하지 말고 `(강사 공지)` 표기**
   ⑤ 피드백 받고 다시 개선하기 — 발급 URL을 친구·가족에게 공유, 받은 의견을 AI에게 전달해 재개선,
      재업로드하면 새 URL이 나온다는 점 명시
   ⑥ 규칙 한 줄 — 업로드는 html 1개, 1MB 이하
3. **레포 운영 내부 사항(.agent/, work-orders, infra 상세)은 언급하지 않는다** — 수강생 무관.
   단, 맨 아래 "운영자 문서" 링크 한 줄로 html-delivery/README.md·infra/README.md 연결은 허용.
4. 게임 파일·기존 문서 수정 금지 — 신규 README.md 1개만.
