# WO-001: run-game 횡스크롤 달리기 게임 ver1 제작
상태: 대기
작성: Claude (Planner) / 실행: Hermes (Coder)
워크트리 게이트: `wo/001` 브랜치 (README 규칙)

## 목표
수강생 실습 소재 2호. `box-game/game-ver1.html`(박스 피하기)과 같은 철학의
횡스크롤 달리기 게임을 `run-game/game-ver1.html`로 만든다.
근거: `docs/planning/DECISIONS.md` — 게임은 수강생이 자기 AI 에이전트와 협업해
스스로 개선하는 출발점이므로 **의도적으로 단순하게** 만든다.

## 설계 결정 (변경 금지)
1. **단일 html 파일, 의존성 제로** — 외부 라이브러리·빌드 도구·이미지 파일 금지. 브라우저에서 바로 열림.
2. **최상단 `CONFIG` 블록** — box-game과 동일하게 `// ===== 설정 (자유롭게 수정하세요!) =====`
   주석으로 감싼 상수 객체. 플레이어 크기/색/점프력, 중력, 장애물 크기/색/속도/생성 간격, 지면 높이 등.
3. **게임 규칙**: Chrome 공룡 게임류 횡스크롤 러너 — 플레이어는 왼쪽 고정 위치에서 달리고,
   장애물이 오른쪽에서 왼쪽으로 접근, 점프(스페이스/↑/터치)로 회피. 충돌 시 게임 오버.
   점수는 넘은 장애물 수 또는 경과 프레임 기반 — 단순한 쪽 선택.
4. **의도적 단순함 유지** — box-game ver1과 같은 수준의 "개선 여지"를 남긴다:
   delta-time 없이 rAF 프레임 기반, 난이도 고정, 최고점수 저장 없음, 일시정지 없음,
   효과음 없음. **완성도를 높이지 말 것** — 이 여백이 수강생의 실습 재료다.
5. **box-game과 동일한 룩앤필·구조** — 다크 배경, 시작/게임오버 UI 오버레이, `점수:` 표시,
   한국어 UI·한국어 주석, 함수 단위 분리(함수 20줄 이하), 전체 200줄 내외.
